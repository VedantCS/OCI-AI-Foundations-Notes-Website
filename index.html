AI Foundations

Brief summary: Core concepts of AI, AGI, ML, DL, types of learning, and generative AI, with real-world examples and use cases.

  • Ability of machines to imitate human intelligence and problem-solving.
  • Includes tasks like learning, reasoning, communication, and creativity.
  • Field that aims to make machines perform tasks that require human intelligence.
  • Example: A self-driving car

  • Machines that fully replicate human capabilities.
  • Can perform complex tasks without human intervention.
  • AI = AGI applied to specific, narrow tasks.

Technologies:
  • Machine Learning
  • Deep Learning
  • Data Science
Domains & Examples:
  • Language: Translation
  • Vision: Image classification
  • Speech: Text to speech
  • ...and many more

  • Definition: A subset of AI that enables machines to learn from data and make predictions/decisions without being explicitly programmed.
  • Example: Spam filter that learns from email content and user actions.
What is an Algorithm in ML?
  • A set of mathematical rules or procedures the model uses to learn from data.

  • Definition: A subset of ML using deep neural networks to learn complex patterns in data.
  • Example: Image recognition software that detects cats in pictures.

Supervised Learning
  • Learns from labelled data (input + known output).
  • Uses labelled data to train algorithms to predict outcomes or classify data.
  • Example: Credit card approval system trained on historical approval data.
  • Process: Input data + known outcomes → model is trained → used to predict outcomes for new data.
Unsupervised Learning
  • Works with unlabelled data to find hidden patterns or groupings (clusters).
  • Example: Retail - grouping customers by income, family size, etc.
Reinforcement Learning
  • Learns through trial and error by interacting with an environment and receiving feedback (reward/punishment).
  • Focuses on training agents to make sequences of decisions through trial and error to maximize rewards.
  • Example: Games like Chess, Tesla Car.

  • Uses layers of neurons (like brain cells) to extract features from raw data (like pixels).
  • Makes decisions without human-defined rules.
  • Best for: Complex tasks like image classification (cat vs dog).
  • Technique: Supervised learning + functional approximation (estimating hidden functions from known data).

  • Definition: A subset of ML focused on creating new content (text, image, audio, etc.).
  • How it works: Learns patterns from existing data and generates new outputs.
  • Example: ChatGPT generates human-like text responses.

ML Foundations

Core ML concepts: data, types, terms, and classic supervised algorithms—clearly presented and easy to reference.

  • Input Data: Features used to describe objects (e.g., colour, texture, eye colour).
  • Label: The correct classification (e.g., "dog" or "cat").
  • Training: Feeding input features and labels to a model to help it learn relationships.
  • Inference: Using the trained model to make predictions on new data.

Supervised Learning
  • Uses: Labelled data
  • Goal: Learn the relationship between features and known labels
  • Applications: Disease detection, Weather forecasting, Stock price prediction, Spam detection, Credit scoring
Unsupervised Learning
  • Uses: Unlabelled data
  • Goal: Discover patterns, groupings, or anomalies
  • Applications: Fraud detection, Segmentation, Outlier detection, Targeted marketing
Reinforcement Learning
  • Uses: Learns by trial and error
  • Goal: Maximize outcomes over time
  • Applications: Self-driving cars
  • How It Works:
    • Define Environment, States, Actions, Rewards/Penalties
    • Training: exploration to exploitation

  • Training Data: Dataset used to teach the model.
  • Trained Model: Learns from data and makes predictions.
  • Prediction/Inference: Model output when given new input.

  • Continuous Output: Use Regression (Linear)
  • Categorical Output: Use Classification (Logistic)
  • Continuous: value within a range (like stock price).
  • Categorical: discrete outcomes ("yes"/"no").

Classification
  • Binary: Two outcomes (e.g., Spam or Not Spam)
  • Multi-class: More than two (e.g., Sentiment: Positive/Negative/Neutral)
Logistic Regression
  • Predicts class probability
  • Uses sigmoid function (values 0-1)
  • Example (Pass/Fail):
    • Feature: Study hours
    • Output: Pass (>0.5), Fail (≤0.5)

Linear regression
  • Models relationship between continuous inputs and outputs (straight line prediction)
  • Adjusts slope (w) and bias (b) to reduce loss
  • Common use: house price prediction
Example: House price prediction
  • Input: House size
  • Output: House price
  • Each pair (size, price) is a training example
  • Scatter plot shows trend. f(x)=w*x+b
  • Learn best fit with loss = (yActual–yPredicted)2
  • Training: learn best line; Prediction: apply to new data

DL Foundations

Major Deep Learning concepts: neural networks, sequence models, architectures, applications, and limitations—clearly presented for deep understanding.

  • Subset of Machine Learning (ML) using Artificial Neural Networks (ANNs) to extract features from raw data.
  • No manual feature extraction required.
  • Supports parallel processing for large datasets, enabling scalability and performance.

  • Modelled after the human brain; composed of neurons (nodes) connected via weights.
  • Neurons compute weighted sums, apply an activation function, and pass results forward.
  • Network Structure:
    • Input Layer: Raw data input (e.g., image pixels).
    • Hidden Layers: Extract internal features.
    • Output Layer: Final predictions (e.g., digit 0–9).
    • Bias: Allows flexibility in learning.
    • Weights: Strength of neuron connections.
Training an ANN:
  • Uses large labeled datasets (e.g., images).
  • Backpropagation algorithm adjusts weights to reduce prediction error iteratively.
  • Model training involves showing data, comparing predictions with true labels, and updating weights for better accuracy.
Example: Digit Recognition Using ANN
  • Input: 28x28 pixel images (784 inputs).
  • Hidden layers: 2 layers, 16 neurons each.
  • Output layer: 10 neurons (digits 0–9).
  • Training process: show image, compute prediction, compare with label, use backpropagation to adjust weights, repeat.

  • Images: Classification, object detection, segmentation → CNNs
  • Text: Translation, sentiment analysis, summarization → Transformers, RNNs, LSTMs
  • Audio: Speech-to-text, music generation → Transformers or sequence models
  • Generative Tasks: Text-to-image, text generation → Transformers, GANs, Diffusion Models

  • Feedforward Neural Network (FNN/MLP): Simplest NN, no memory.
  • Convolutional Neural Network (CNN): Designed for image/video data, learns patterns automatically.
  • Recurrent Neural Network (RNN): Handles sequential data, maintains hidden state.
  • Autoencoders: Unsupervised, for feature extraction, compression, anomaly detection.
  • Long Short-Term Memory (LSTM): Solves long-term sequence dependencies.
  • Generative Adversarial Networks (GANs): Generates realistic synthetic data.
  • Transformers: SOTA for NLP and generative tasks.

Sequence Models
  • Sequences: ordered data points/events (e.g., time series).
  • Goal: Detect temporal patterns for prediction/classification/generation.
Recurrent Neural Networks (RNNs):
  • Feedback loops, maintains hidden state across time steps.
  • Architectures:
    • One-to-One: Like feedforward NN.
    • One-to-Many: e.g., music generation.
    • Many-to-One: e.g., sentiment analysis.
    • Many-to-Many: e.g., machine translation.
  • Limitation: Vanishing gradient in long-term dependencies; LSTM helps overcome this.
Long Short-Term Memory (LSTM):
  • Captures long-term dependencies with memory cells and gating mechanisms.
  • Components:
    • Input Gate: Stores new info.
    • Forget Gate: Discards old info.
    • Output Gate: Determines output from current memory.
  • How LSTM works:
    • Input, previous hidden and cell state.
    • Gates filter/update cell state, produce new hidden state for next step.

What is CNN?
  • Designed for grid-like data (images/videos), processes 2D data directly.
  • Goal: extract key features while reducing data size.
CNN Architecture Overview
  1. Input Layer: Takes image data (2D).
  2. Feature Extraction Layers: Repeated blocks of
    • Convolutional Layer: Applies filters (edges/corners).
    • Activation Function (ReLU): Introduces non-linearity.
    • Pooling Layer: Downsamples, speeds computation.
  3. Classification Layers:
    • Fully Connected Layers: Final predictions.
    • Softmax Layer: Probability scores.
    • Dropout Layer: Prevents overfitting.

  • High computational cost for large datasets.
  • Risk of overfitting with small/imbalanced datasets.

Gen AI / LLMs

Essentials of Generative AI, Large Language Models, and Transformer architectures—concepts, processes, and key mechanism summaries.

  • Subset of deep learning focused on creating new content (not just classifying or predicting).
  • Learns patterns from large datasets, generates new data based on patterns (e.g., new images after learning from others).
  • Does not copy—creates original content.
  • Uses unlabelled data during pretraining.
Types of Generative AI Models:
  • Text-based GenAI: Generates text, code, dialogue, articles. Learns from large text datasets.
  • Multimodal GenAI: Processes and generates text, images, audio, video—multiple media types.
Key Terms:
  • Training: Learning from data.
  • Inference: Predictions/generating outputs from trained models.
  • Supervised Learning: Learning from labelled data.
  • Fine-tuning: Adapting a pre-trained model for a specific task with labelled data.

LLM Definition:
  • Probabilistic model that calculates likelihood of word sequences.
  • Predicts next word based on previous words.
  • Precise: Deep learning model based on transformer architecture, trained on huge text datasets for generating human-like text.
“Large” in LLMs:
  • Millions to hundreds of billions of parameters.
  • More parameters = higher capacity and performance (up to a point).
How LLMs Work (Simplified):
  1. Take sequence of words.
  2. Compute probabilities for next word.
  3. Select most likely word, append to sequence.
  4. Repeat until EOS (end-of-sequence) token.
  5. Learning occurs by adjusting weights (parameters) during training to predict next word.
Notes:
  • Too many parameters can cause overfitting (good on training data, poor on new data).
  • Performance also depends on data quality and model optimization.

Definition:
  • Transformer: Processes entire input with self-attention, capturing context & relationships between all words.
Why Transformers?
  • Parallel processing—faster than RNNs, can "see" the entire sequence at once.
  • Solves long-range dependency problem, enables better context.
  • Powers modern LLMs (ChatGPT, etc).
Architecture Overview:
  • Encoder: Converts input to contextual embeddings.
  • Decoder: Generates output (text), uses encoder output for context.
  • Multiple encoder/decoder layers use self-attention for understanding relationships.
Self-Attention Mechanism:
  • Weighs importance of each word in context.
  • Connects all words, not just nearby ones.
  • Helps answer: “Which other words should I focus on for context?”

Tokenization:
  • LLMs operate on tokens (words or parts of words, punctuation).
  • Simple text: ~1 token/word; complex: ~2–3 tokens/word.
Embeddings:
  • Numeric/vector representations of tokens, phrases, sentences.
  • Allow models to understand meaning & semantic similarity.
  • Use cases: classification, vector databases, semantic search.
Retrieval-Augmented Generation (RAG):
  • Enhances LLMs with external knowledge from vector retrieval (search + generation).
  • How RAG works:
    1. Input query encoded as embeddings.
    2. Matched to documents from a vector database.
    3. Content sent to LLM—uses both retrieved content and its own knowledge.
    4. Embeddings are crucial for retrieval quality and reducing hallucinations.

  • Decoder Models: Generate next token one at a time, repeat until EOS token.
  • Encoder-Decoder Models: Use encoder for input, decoder for output generation.
  • These architectures power translation, summarization, and sequential text generation.

Prompt Engineering

A guide to structuring inputs for LLMs, best practices, techniques, and key challenges.

  1. Start with a simple prompt.
  2. Add few-shot prompting.
  3. Add simple retrieval via RAG.
  4. Fine-tune the model.
  5. Optimize retrieval on fine-tuned model.

  • Prompt: Input text for a desired LLM response.
  • Prompt Engineering: Designing inputs for more useful LLM outputs.
  • LLMs predict the next token in a sequence (“completion-based learning”).
In-Context Learning:
  • Zero-shot: Question only.
  • One-shot: One example.
  • Few-shot: Multiple examples.
  • Model does not update internal weights, but “learns” from the prompt structure.
Chain-of-Thought Prompting:
  • Prompt LLM to “show its work” stepwise for complex reasoning.
  • Example: 5 balls + (2 cans × 3 balls) → Model response: 2 × 3 = 6, 5 + 6 = 11.

Hallucination: Fluent but incorrect/made-up output from LLMs.

GPUs and Superclusters in OCI

OCI's RDMA Supercluster: Key Takeaways

OCI uses Remote Direct Memory Access (RDMA) technology to enable high-speed, low-latency data transfer by bypassing the CPU. The RDMA Supercluster uses RDMA over Converged Ethernet (RoCE) to support massive GPU workloads, built on a three-tier Clos fabric architecture for optimal performance.

  • Lossless Networking: Tuned buffers prevent packet loss with latency ~20 microseconds.
  • Intelligent Placement: Control plane optimizes placement of latency-sensitive workloads.
  • Network Locality Hints: Ensures traffic locality to reduce latency and collisions.

AI Vector Search lets databases store numerical embeddings for semantic retrieval of diverse data types, supporting complex GenAI pipelines within Oracle's converged database architecture.

How it Works

  • Vector Generation: Models (including ONNX format) convert data to vectors, loaded via VECTOR_EMBEDDING SQL function.
  • Similarity Search: VECTOR_DISTANCE function measures similarity; smaller distance = closer match; query across multiple tables and join with classic SQL.
  • Performance & Accuracy: Vector indexes speed searches, with options for in-memory, partitioning, and TARGET ACCURACY for approximate result control. Use APPROXIMATE keyword for optimized index queries.

AI Vector Search underpins Retrieval-Augmented Generation (RAG) pipelines, enhancing LLM output with relevant data and context.

  1. Data Sources: Load structured and unstructured data (databases, files, social media).
  2. Transformation & Vectorization: Documents are transformed, split, summarized, and embedded as vectors stored in the database.
  3. Search & RAG: User queries are converted to vectors for similarity search; retrieved documents help augment LLM prompts for more accurate, context-aware answers.

This integration allows advanced operations—like matching job positions semantically and filtering by city—using a single SQL query. Oracle Database 23ai also integrates with frameworks such as LangChain and LlamaIndex for developer flexibility.
